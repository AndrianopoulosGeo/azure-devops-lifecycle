# /setup-pipeline — Azure DevOps CI/CD Pipeline Setup

> **Expert Voice:** DevOps Engineer — builds CI/CD pipelines, configures service connections, manages variable groups, and ensures automated deployments work end-to-end.

You are a DevOps Engineer setting up a complete Azure DevOps CI/CD pipeline for a project. Your job is to create service connections, variable groups, environments with approvals, generate pipeline YAML files, register the pipeline, authorize resources, and verify the first run succeeds.

## Prerequisites

Before starting, verify:
1. `.env.claude` exists with valid configuration
2. `.env.infra` exists (generated by `/setup-infra`)
3. `.env` exists with application secrets
4. Azure CLI (`az`) is installed and authenticated
5. Dockerfile exists in the repository root

If any prerequisite is missing, stop and provide clear instructions to fix it.

### 1.2 Check Hetzner Agent Online

Configure Azure DevOps CLI defaults:

```bash
export AZURE_DEVOPS_EXT_PAT=$AZURE_DEVOPS_PAT
az devops configure --defaults organization=https://dev.azure.com/$AZURE_DEVOPS_ORG project="$INFRA_PROJECT"
```

**IMPORTANT:** `az pipelines agent list` does NOT exist. Use `az devops invoke`:

```bash
POOL_ID=$(az devops invoke --area distributedTask --resource pools --query "value[?name=='Hetzner'].id | [0]" -o tsv)
ONLINE_COUNT=$(az devops invoke --area distributedTask --resource agents --route-parameters poolId=$POOL_ID --query "value[?status=='online'] | length(@)" -o tsv)
echo "Online agents: $ONLINE_COUNT"
```

If no agents are online (`ONLINE_COUNT` is 0 or empty):

1. Trigger the `manage-server.yaml` pipeline in `$INFRA_PROJECT`:
   ```bash
   az pipelines run --name "manage-server" --project "$INFRA_PROJECT"
   ```
2. Poll every 30 seconds for up to 5 minutes:
   ```bash
   for i in $(seq 1 10); do
     sleep 30
     ONLINE_COUNT=$(az devops invoke --area distributedTask --resource agents --route-parameters poolId=$POOL_ID --query "value[?status=='online'] | length(@)" -o tsv)
     if [ "$ONLINE_COUNT" -gt 0 ]; then
       echo "Agent is online."
       break
     fi
     echo "Waiting for agent... ($((i*30))s elapsed)"
   done
   ```
3. If still offline after 5 minutes, stop and report the error. The pipeline requires the Hetzner agent.

## Phase 3: Create Service Connections (REST API)

**CRITICAL:** Must use REST API directly, NOT `az devops service-endpoint create` (it has bugs with certain auth schemes).

**CRITICAL:** Write JSON to temp files using heredocs. Do NOT use process substitution (`<(...)`) — it fails silently with `az devops invoke`.

Set the base URL for REST calls:

```bash
ORG_URL="https://dev.azure.com/$AZURE_DEVOPS_ORG"
```

### 3.1 ARM Service Connection (Azure Resource Manager)

Create the JSON payload for an ARM service connection using ServicePrincipal scheme:

```bash
cat > /tmp/arm-sc.json <<'ARMEOF'
{
  "name": "ARM-SERVICE-CONNECTION",
  "type": "azurerm",
  "url": "https://management.azure.com/",
  "data": {
    "subscriptionId": "AZURE_SUBSCRIPTION_ID_PLACEHOLDER",
    "subscriptionName": "SUBSCRIPTION_NAME_PLACEHOLDER",
    "environment": "AzureCloud",
    "scopeLevel": "Subscription",
    "creationMode": "Manual"
  },
  "authorization": {
    "scheme": "ServicePrincipal",
    "parameters": {
      "tenantid": "AZURE_TENANT_ID_PLACEHOLDER",
      "serviceprincipalid": "AZURE_CLIENT_ID_PLACEHOLDER",
      "authenticationType": "spnKey",
      "serviceprincipalkey": "AZURE_CLIENT_SECRET_PLACEHOLDER"
    }
  },
  "serviceEndpointProjectReferences": [
    {
      "projectReference": {
        "id": "PROJECT_ID_PLACEHOLDER",
        "name": "AZURE_DEVOPS_PROJECT_PLACEHOLDER"
      },
      "name": "ARM-SERVICE-CONNECTION"
    }
  ]
}
ARMEOF
```

Replace placeholders using `sed` with `|` delimiter (subscription names may contain `/`):

```bash
sed -i "s|AZURE_SUBSCRIPTION_ID_PLACEHOLDER|$AZURE_SUBSCRIPTION_ID|g" /tmp/arm-sc.json
sed -i "s|SUBSCRIPTION_NAME_PLACEHOLDER|$SUBSCRIPTION_NAME|g" /tmp/arm-sc.json
sed -i "s|AZURE_TENANT_ID_PLACEHOLDER|$AZURE_TENANT_ID|g" /tmp/arm-sc.json
sed -i "s|AZURE_CLIENT_ID_PLACEHOLDER|$AZURE_CLIENT_ID|g" /tmp/arm-sc.json
sed -i "s|AZURE_CLIENT_SECRET_PLACEHOLDER|$AZURE_CLIENT_SECRET|g" /tmp/arm-sc.json
sed -i "s|PROJECT_ID_PLACEHOLDER|$PROJECT_ID|g" /tmp/arm-sc.json
sed -i "s|AZURE_DEVOPS_PROJECT_PLACEHOLDER|$AZURE_DEVOPS_PROJECT|g" /tmp/arm-sc.json
```

Create the service connection via REST API:

```bash
ARM_SC_RESPONSE=$(curl -s -X POST \
  -u ":$AZURE_DEVOPS_PAT" \
  -H "Content-Type: application/json" \
  -d @/tmp/arm-sc.json \
  "$ORG_URL/_apis/serviceendpoint/endpoints?api-version=7.1")
ARM_SC_ID=$(echo "$ARM_SC_RESPONSE" | jq -r '.id')
echo "ARM Service Connection ID: $ARM_SC_ID"
```

If `ARM_SC_ID` is null or empty, check if it already exists:

```bash
ARM_SC_ID=$(curl -s -u ":$AZURE_DEVOPS_PAT" \
  "$ORG_URL/$AZURE_DEVOPS_PROJECT/_apis/serviceendpoint/endpoints?endpointNames=ARM-SERVICE-CONNECTION&api-version=7.1" \
  | jq -r '.value[0].id')
echo "Existing ARM Service Connection ID: $ARM_SC_ID"
```

Clean up:

```bash
rm -f /tmp/arm-sc.json
```

## Phase 4: Create Variable Group + Environments

### 4.1 Create Variable Group

The variable group combines values from TWO sources: `.env.infra` (infrastructure) and `.env` (application).

First, create the group with non-secret variables from `.env.infra`:

```bash
az pipelines variable-group create \
  --name "pipeline-variables" \
  --variables \
    AZURE_CLIENT_ID="$AZURE_CLIENT_ID" \
    AZURE_TENANT_ID="$AZURE_TENANT_ID" \
    AZURE_SUBSCRIPTION_ID="$AZURE_SUBSCRIPTION_ID" \
    AZURE_RESOURCE_GROUP="$AZURE_RESOURCE_GROUP" \
    AZURE_ACR_LOGIN_SERVER="$AZURE_ACR_LOGIN_SERVER" \
    AZURE_APP_SERVICE_STAGING="$AZURE_APP_SERVICE_STAGING" \
    AZURE_APP_SERVICE_PRODUCTION="$AZURE_APP_SERVICE_PRODUCTION" \
  --authorize false \
  --output json
```

Get the variable group ID:

```bash
VG_ID=$(az pipelines variable-group list --query "[?name=='pipeline-variables'].id | [0]" -o tsv)
echo "Variable Group ID: $VG_ID"
```

Add the secret from `.env.infra` individually:

```bash
az pipelines variable-group variable create \
  --group-id $VG_ID \
  --name AZURE_CLIENT_SECRET \
  --value "$AZURE_CLIENT_SECRET" \
  --secret true
```

Now add application variables from `.env`. Detect secrets by name pattern:

```bash
while IFS='=' read -r key value; do
  # Skip empty lines and comments
  [[ -z "$key" || "$key" =~ ^# ]] && continue
  # Trim whitespace
  key=$(echo "$key" | xargs)
  value=$(echo "$value" | xargs)
  # Detect if variable is a secret by name
  if echo "$key" | grep -qiE "(SECRET|KEY|PASSWORD|TOKEN|CONNECTION_STRING)"; then
    echo "Adding secret: $key"
    az pipelines variable-group variable create \
      --group-id $VG_ID \
      --name "$key" \
      --value "$value" \
      --secret true
  else
    echo "Adding variable: $key"
    az pipelines variable-group variable create \
      --group-id $VG_ID \
      --name "$key" \
      --value "$value"
  fi
done < .env
```

### 4.3 Production Approval Gate

Pre-compute the approver identity. Get the current user's identity:

```bash
APPROVER_ID=$(curl -s -u ":$AZURE_DEVOPS_PAT" \
  "https://vssps.dev.azure.com/$AZURE_DEVOPS_ORG/_apis/connectionData" \
  | jq -r '.authenticatedUser.id')
APPROVER_NAME=$(curl -s -u ":$AZURE_DEVOPS_PAT" \
  "https://vssps.dev.azure.com/$AZURE_DEVOPS_ORG/_apis/connectionData" \
  | jq -r '.authenticatedUser.providerDisplayName')
echo "Approver: $APPROVER_NAME ($APPROVER_ID)"
```

Try adding the approval check via REST API:

```bash
cat > /tmp/approval-check.json <<APPROVALEOF
{
  "type": {
    "id": "8c3f3133-8639-4154-8e73-b8a58aa11afd",
    "name": "Approval"
  },
  "settings": {
    "approvers": [
      {
        "id": "$APPROVER_ID",
        "displayName": "$APPROVER_NAME"
      }
    ],
    "executionOrder": 1,
    "minRequiredApprovers": 1,
    "instructions": "Approve production deployment"
  },
  "resource": {
    "type": "environment",
    "id": "$PRODUCTION_ENV_ID"
  }
}
APPROVALEOF

APPROVAL_RESPONSE=$(curl -s -o /dev/null -w "%{http_code}" -X POST \
  -u ":$AZURE_DEVOPS_PAT" \
  -H "Content-Type: application/json" \
  -d @/tmp/approval-check.json \
  "$ORG_URL/$AZURE_DEVOPS_PROJECT/_apis/pipelines/checks/configurations?api-version=7.1-preview.1")
rm -f /tmp/approval-check.json
echo "Approval API response: $APPROVAL_RESPONSE"
```

If the REST API fails (response is not `200` or `201`), fall back to Playwright:

> "REST API for approval checks failed. Falling back to browser automation to configure the production approval gate."

Use Playwright to configure the approval:

1. Navigate to `$ORG_URL/$AZURE_DEVOPS_PROJECT/_environments/$PRODUCTION_ENV_ID?view=checks`
2. Click "Add check" or the "+" button
3. Select "Approvals" from the check type list
4. Add the current user as approver in the approvers field
5. Click "Create" to save the approval check

If Playwright also fails, warn the user:
> "Could not configure production approval automatically. Please add an approval check manually: Navigate to Project Settings > Environments > production > Add check > Approvals."

## Phase 5: Generate Pipeline YAML

All pipelines use `pool: Hetzner` and Linux-native `script:` steps (not PowerShell).

### 5.1 Create Directory Structure

```bash
mkdir -p pipelines/variables pipelines/templates scripts
```

### 5.2 Orchestrator: `azure-pipelines.yml`

Generate the main pipeline file at the repository root. The pipeline has these stages:
- **Build** — Docker build only (no push)
- **Lint** — Tech-stack-specific linting
- **UnitTests** — Run unit tests via docker compose
- **E2ETests** — Run E2E tests via docker compose
- **PushACR** — Push built image to ACR (conditional: only on staging or main branches)
- **DeployStaging** — Deploy to staging App Service (conditional: only on staging branch)
- **DeployProduction** — Deploy to production App Service (conditional: only on main branch)

**CRITICAL:** PushACR is push-only, NOT a rebuild. The Hetzner agent retains Docker layer cache, so the image built in the Build stage is still available.

Write `azure-pipelines.yml`:

```yaml
trigger:
  branches:
    include:
      - develop
      - staging
      - main

variables:
  - template: pipelines/variables/common.yml
  - group: pipeline-variables

stages:
  - stage: Build
    displayName: 'Build'
    pool:
      name: Hetzner
    jobs:
      - job: Build
        steps:
          - template: pipelines/templates/build.yml

  - stage: Lint
    displayName: 'Lint'
    dependsOn: Build
    pool:
      name: Hetzner
    jobs:
      - job: Lint
        steps:
          - template: pipelines/templates/lint.yml

  - stage: UnitTests
    displayName: 'Unit Tests'
    dependsOn: Build
    pool:
      name: Hetzner
    jobs:
      - job: UnitTests
        steps:
          - template: pipelines/templates/unit-tests.yml

  - stage: E2ETests
    displayName: 'E2E Tests'
    dependsOn: UnitTests
    pool:
      name: Hetzner
    jobs:
      - job: E2ETests
        steps:
          - template: pipelines/templates/e2e-tests.yml

  - stage: PushACR
    displayName: 'Push to ACR'
    dependsOn:
      - Lint
      - UnitTests
      - E2ETests
    condition: and(succeeded(), or(eq(variables['Build.SourceBranch'], 'refs/heads/staging'), eq(variables['Build.SourceBranch'], 'refs/heads/main')))
    pool:
      name: Hetzner
    jobs:
      - job: PushACR
        steps:
          - script: |
              echo "Logging in to ACR..."
              docker login $(acrLoginServer) -u $(AZURE_CLIENT_ID) -p $(AZURE_CLIENT_SECRET)
              echo "Pushing image..."
              docker push $(acrLoginServer)/$(imageRepository):$(tag)
            displayName: 'Push image to ACR'

  - stage: DeployStaging
    displayName: 'Deploy to Staging'
    dependsOn: PushACR
    condition: and(succeeded(), eq(variables['Build.SourceBranch'], 'refs/heads/staging'))
    pool:
      name: Hetzner
    jobs:
      - deployment: DeployStaging
        environment: staging
        strategy:
          runOnce:
            deploy:
              steps:
                - template: pipelines/templates/deploy.yml
                  parameters:
                    environment: staging
                    appServiceName: $(AZURE_APP_SERVICE_STAGING)

  - stage: DeployProduction
    displayName: 'Deploy to Production'
    dependsOn: PushACR
    condition: and(succeeded(), eq(variables['Build.SourceBranch'], 'refs/heads/main'))
    pool:
      name: Hetzner
    jobs:
      - deployment: DeployProduction
        environment: production
        strategy:
          runOnce:
            deploy:
              steps:
                - template: pipelines/templates/deploy.yml
                  parameters:
                    environment: production
                    appServiceName: $(AZURE_APP_SERVICE_PRODUCTION)
```

### 5.8 Deploy Template: `pipelines/templates/deploy.yml`

Parameterized template for both staging and production:

```yaml
parameters:
  - name: environment
    type: string
  - name: appServiceName
    type: string

steps:
  - script: |
      chmod +x scripts/generate-env.sh
      ./scripts/generate-env.sh
    displayName: 'Generate .env file from pipeline variables'
  - script: |
      az login --service-principal \
        -u $(AZURE_CLIENT_ID) \
        -p $(AZURE_CLIENT_SECRET) \
        --tenant $(AZURE_TENANT_ID)
    displayName: 'Azure CLI login'
  - script: |
      az webapp config container set \
        --name ${{ parameters.appServiceName }} \
        --resource-group $(AZURE_RESOURCE_GROUP) \
        --container-image-name $(acrLoginServer)/$(imageRepository):$(tag) \
        --container-registry-url https://$(acrLoginServer) \
        --container-registry-user $(AZURE_CLIENT_ID) \
        --container-registry-password $(AZURE_CLIENT_SECRET)
    displayName: 'Configure container on App Service'
  - script: |
      az webapp restart \
        --name ${{ parameters.appServiceName }} \
        --resource-group $(AZURE_RESOURCE_GROUP)
    displayName: 'Restart App Service'
```

### 5.9 Environment Script: `scripts/generate-env.sh`

This script reconstructs the `.env` file from pipeline variables at runtime. Generate it dynamically based on the actual variable names in the project's `.env` file:


## Phase 6: Create Pipeline + Authorize Resources

### 6.1 Create Pipeline in Azure DevOps

```bash
PIPELINE_RESPONSE=$(az pipelines create \
  --name "$AZURE_DEVOPS_REPO" \
  --repository "$AZURE_DEVOPS_REPO" \
  --repository-type tfsgit \
  --branch develop \
  --yml-path azure-pipelines.yml \
  --skip-first-run true \
  --output json)
PIPELINE_ID=$(echo "$PIPELINE_RESPONSE" | jq -r '.id')
echo "Pipeline ID: $PIPELINE_ID"
```

### 6.2 Authorize ALL Resources

The pipeline needs access to 6 resources. Authorize each via REST API PATCH.

**CRITICAL:** Authorize with BOTH `allPipelines: true` AND a specific `pipelines[{id}]` entry. Some resources require both.

First, get the agent pool queue ID:

```bash
QUEUE_ID=$(az devops invoke --area distributedTask --resource pools --query "value[?name=='Hetzner'].id | [0]" -o tsv)
echo "Hetzner Queue ID: $QUEUE_ID"
```

**Authorize Agent Pool (Hetzner):**

```bash
curl -s -X PATCH \
  -u ":$AZURE_DEVOPS_PAT" \
  -H "Content-Type: application/json" \
  -d "{\"pipelines\": [{\"id\": $PIPELINE_ID, \"authorized\": true}], \"resource\": {\"id\": \"$QUEUE_ID\", \"type\": \"queue\"}, \"allPipelines\": {\"authorized\": true}}" \
  "$ORG_URL/$AZURE_DEVOPS_PROJECT/_apis/pipelines/pipelinepermissions/queue/$QUEUE_ID?api-version=7.1-preview.1"
```

**Authorize ARM Service Connection:**

```bash
curl -s -X PATCH \
  -u ":$AZURE_DEVOPS_PAT" \
  -H "Content-Type: application/json" \
  -d "{\"pipelines\": [{\"id\": $PIPELINE_ID, \"authorized\": true}], \"resource\": {\"id\": \"$ARM_SC_ID\", \"type\": \"endpoint\"}, \"allPipelines\": {\"authorized\": true}}" \
  "$ORG_URL/$AZURE_DEVOPS_PROJECT/_apis/pipelines/pipelinepermissions/endpoint/$ARM_SC_ID?api-version=7.1-preview.1"
```

**Authorize ACR Service Connection:**

```bash
curl -s -X PATCH \
  -u ":$AZURE_DEVOPS_PAT" \
  -H "Content-Type: application/json" \
  -d "{\"pipelines\": [{\"id\": $PIPELINE_ID, \"authorized\": true}], \"resource\": {\"id\": \"$ACR_SC_ID\", \"type\": \"endpoint\"}, \"allPipelines\": {\"authorized\": true}}" \
  "$ORG_URL/$AZURE_DEVOPS_PROJECT/_apis/pipelines/pipelinepermissions/endpoint/$ACR_SC_ID?api-version=7.1-preview.1"
```

**Authorize Variable Group:**

```bash

  -u ":$AZURE_DEVOPS_PAT" \
```

**Authorize Staging Environment:**

## Phase 7: Trigger + Verify

### 7.1 Trigger Pipeline

```bash
RUN_RESPONSE=$(az pipelines run --name "$AZURE_DEVOPS_REPO" --branch develop --output json)
RUN_ID=$(echo "$RUN_RESPONSE" | jq -r '.id')
echo "Pipeline run ID: $RUN_ID"
echo "Pipeline URL: $ORG_URL/$AZURE_DEVOPS_PROJECT/_build/results?buildId=$RUN_ID"
```

### 7.2 Poll for Completion

Poll every 20 seconds for up to 10 minutes:

```bash
for i in $(seq 1 30); do
  sleep 20
  STATUS=$(az pipelines runs show --id $RUN_ID --query "status" -o tsv)
  RESULT=$(az pipelines runs show --id $RUN_ID --query "result" -o tsv)
  echo "Poll $i: status=$STATUS result=$RESULT"
  if [ "$STATUS" = "completed" ]; then
    break
  fi
done
```

### 7.3 Report Results

**If the pipeline passed** (`RESULT` is `succeeded`):

Report which stages ran. On the develop branch, Build, Lint, UnitTests, and E2ETests should run, but PushACR and Deploy stages should be skipped (they only run on staging/main):

```
Pipeline run #$RUN_ID: SUCCEEDED

Stages executed:
  [PASS] Build
  [PASS] Lint
  [PASS] Unit Tests
  [PASS] E2E Tests
  [SKIP] Push to ACR (develop branch — skipped as expected)
  [SKIP] Deploy to Staging (develop branch — skipped as expected)
  [SKIP] Deploy to Production (develop branch — skipped as expected)
```

**If the pipeline failed** (`RESULT` is `failed`):

Retrieve the build logs and present common fixes:

```bash
az pipelines runs show --id $RUN_ID --output json
```

Common issues and fixes:
- **Agent not found**: Check Hetzner agent is online, pool name matches `Hetzner`
- **Docker build fails**: Check Dockerfile syntax, missing build context files
- **Variable group not authorized**: Re-run the Phase 6 authorization commands
- **Environment not authorized**: Re-run the Phase 6 authorization commands
- **Lint fails**: Fix linting issues in the code, then push and re-run

### 7.4 Update Pipeline IDs in .env.claude

```bash
sed -i "s|^STAGING_PIPELINE_ID=.*|STAGING_PIPELINE_ID=$PIPELINE_ID|" .env.claude
sed -i "s|^PRODUCTION_PIPELINE_ID=.*|PRODUCTION_PIPELINE_ID=$PIPELINE_ID|" .env.claude
```
