# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

A Claude Code plugin (`azure-devops-lifecycle`) that provides 15 slash commands for full development lifecycle automation with Azure DevOps and Gitflow branching. Distributed via a custom marketplace (`AndrianopoulosGeo/claude-marketplace`).

## Architecture

### Plugin Structure

```
.claude-plugin/plugin.json    — Plugin metadata, version, marketplace registry
commands/                     — Slash command definitions (each .md = one command)
commands/_shared/             — Reusable fragments sourced by multiple commands
templates/                    — Files copied into consuming projects by /init-project
evals/                        — Eval specs for testing command behavior and trigger accuracy
docs/plans/                   — Implementation plans for this plugin's own development
```

### How Commands Work

Each `commands/*.md` file is a Claude Code slash command. The frontmatter defines `allowed-tools` and `description`. The body is a step-by-step prompt that Claude follows when the command is invoked.

Commands are **not code** — they're structured prompts with bash snippets, decision logic, and references to other skills/plugins.

### Shared Fragments

`commands/_shared/` contains reusable patterns that commands reference (not import — they're read inline):
- `load-config.md` — `.env.claude` loading and Azure DevOps auth (used by all lifecycle commands)
- `state-management.md` — `.state.md` read/write patterns and track→step→next mappings

### Command Dependency Chain

Commands delegate to each other and to external plugins:

| Command | Delegates To |
|---------|-------------|
| `/commit` | `commit-commands:commit` (external plugin) |
| `/develop`, `/quick-fix`, `/hotfix` | `/commit` for all commits |
| `/develop` | `superpowers:executing-plans`, `pr-review-toolkit:code-simplifier`, `pr-review-toolkit:review-pr` |
| `/quick-fix`, `/hotfix` | `superpowers:test-driven-development`, `superpowers:verification-before-completion` |
| `/feature` | `superpowers:brainstorming`, `superpowers:writing-plans` |
| `/next` | Reads `.state.md` and invokes the appropriate next command |
| `/setup-infra` | (standalone — Azure CLI only) |
| `/setup-pipeline` | `/setup-infra` for `.env.infra`, Playwright (fallback for approval gate) |

### Project Setup (one-time)

```
/init-project → /setup-infra → /setup-pipeline
```

### Workflow Tracks

Three tracks flow through commands in sequence, tracked by `.state.md`:

```
Feature:   /feature → /develop → /staging → /release
Quick-fix: /quick-fix → /staging → /release
Hotfix:    /hotfix (direct to master, then backmerge to develop)
```

## Key Conventions

### Commit Rules (enforced by /commit)

- Conventional Commits format: `<type>(<scope>): <summary>`
- **No AI attribution** — never include `Co-Authored-By`, `Generated by`, Claude, Anthropic, or any AI reference in commit messages
- Auto-update context docs (`CLAUDE.md`, `docs/architecture.md`, `docs/TESTING.md`, `docs/wiki/`) when code changes warrant it

### Configuration

All commands read from `.env.claude` (never hardcoded values). Required fields: `AZURE_DEVOPS_ORG`, `AZURE_DEVOPS_PROJECT`, `AZURE_DEVOPS_PAT`, `DEPLOY_TARGET`, `TECH_STACK`.

For Azure deployments, `/setup-infra` generates `.env.infra` with Service Principal credentials and resource names. `/setup-pipeline` reads both files.

### State Management

Lifecycle commands read/write `.state.md` to track workflow progress. Status values: `idle`, `in-progress`, `blocked`, `awaiting-review`, `ready-to-promote`.

## Evals

Eval specs live in `evals/` as JSON files. Two types:
- **Trigger evals** (`trigger-evals.json`) — test that the right command is invoked for a given prompt
- **Execution evals** (`*-evals.json`) — test that a command produces expected output sections

Run evals via the `skill-creator` plugin: `/skill-creator run evals for <command> using evals/<file>.json`

## Version & Distribution

- Version is in `.claude-plugin/plugin.json` — bump on every release
- Source repo: Azure DevOps (`origin`) for development, GitHub (`github`) for marketplace distribution
- Marketplace version is in a separate repo (`AndrianopoulosGeo/claude-marketplace`) in `.claude-plugin/marketplace.json` — must be bumped separately after pushing to GitHub
- **Both remotes must be pushed** for consumers to get updates
